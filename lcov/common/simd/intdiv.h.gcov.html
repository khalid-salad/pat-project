      14             :  * Almost all architecture (except Power10) doesn't support integer vector division,
      15             :  * also the cost of scalar division in architectures like x86 is too high it can take
      16             :  * 30 to 40 cycles on modern chips and up to 100 on old ones.
      17             :  *
      18             :  * Therefore we are using division by multiplying with precomputed reciprocal technique,
      19             :  * the method that been used in this implementation is based on T. Granlund and P. L. Montgomery
      20             :  * “Division by invariant integers using multiplication(see [Figure 4.1, 5.1]
      21             :  * http://citeseerx.ist.psu.edu/viewdoc/summary?doi=10.1.1.1.2556)
      22             :  *
      23             :  * It shows a good impact for all architectures especially on X86,
      24             :  * however computing divisor parameters is kind of expensive so this implementation
      25             :  * should only works when divisor is a scalar and used multiple of times.
      26             :  *
      27             :  * The division process is separated into two intrinsics for each data type
      28             :  *
      29             :  *  1- npyv_{dtype}x3 npyv_divisor_{dtype} ({dtype} divisor);
      30             :  *     For computing the divisor parameters (multiplier + shifters + sign of divisor(signed only))
      31             :  *
      32             :  *  2- npyv_{dtype} npyv_divisor_{dtype} (npyv_{dtype} dividend, npyv_{dtype}x3 divisor_parms);
